# 國際交流成效回報系統 — 規格書

## 目標
取代 Excel / Google 表單，讓各系所線上回報「學生國際交流成效」，國際處後台自動彙整三項指標並可匯出 Excel。

## 三項指標
1. **出國交流人次**：大類為「出國交流」的活動人數總和（移地教學、文化交流團、國際參訪、境外實習、境外工作營、國際競賽及展演）。
2. **研討會/工作營/工作坊場次**：大類為「研討會工作營工作坊」的活動筆數。
3. **研討會/工作營/工作坊人數**：同上大類的人數總和。

## 角色與存取
- **系所填報者**：免登入。從入口選學院→系所，輸入姓名後填寫。
- **學院**：免登入、唯讀。學院頁顯示全院彙整與各系狀態（不審核、不編輯）。
- **國際處 admin**：Supabase Auth 登入，看全校、退回解鎖、匯出 Excel、管理期別。

## 安全模型（重點）
- 填報端所有讀寫一律經 Next.js API（server 端以 `service_role` 執行，繞過 RLS）。
- 瀏覽器只拿到 `anon` key，且僅供 admin 登入；資料表 RLS 預設拒絕 anon。
- `service_role` 僅存在於伺服器環境變數，永不外流到 client。

## 頁面流程
1. `/` 入口：依校區分組的學院卡片。
2. `/college/[college]` 學院頁：三指標彙整 + 各系狀態表（唯讀）+ 各系「填寫」連結。
3. `/dept/[unitId]` 系填報頁：
   - 進入前要求「填報人姓名」（取代登入身分，記錄於每筆）。
   - **全系共用清單**：顯示本系本期所有活動（不分填報人）；送出前任何人皆可新增/編輯/刪除。
   - 大類→類型連動下拉；人數限數字。
   - 即時顯示本系三指標小計。
   - 「確認送出」→ 以（系×期別）為單位鎖定；或「申報本期無活動」。
4. `/admin`：登入後總覽、各系狀態、退回、匯出 Excel、切換/管理期別。

## 規則
- 送出後鎖定，系所無法再改；需 admin「退回」才解鎖。
- 沒活動的系所可「申報本期無活動」；後台據此區分「未填 / 已申報無活動」。
- 系統以「期別」運作，可每學期重用（periods 表，建議同時只開一期 is_open=true）。

## 資料表
periods / units(42 系所，含國際事務處) / activities / submissions。
彙總視圖 `v_activity_metrics`（period × unit 的三指標）。詳見 `db/schema.sql`。
